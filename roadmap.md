3a. [x] Clean the old attention-gap measurement strategy — closed. All four criteria met: (1) no confounded L_inf=127/255 claims remain in code or output — the only 127/255 references are legitimate i8 quantization bounds; (2) `corridor.rs:measure_corridor()` fails closed ("no prefix_retained", "no prefix_shell_openings", "no shell_opening", "empty prefix_shell_openings") when required data is missing; (3) naming cleanly distinguishes `measure_attention_diff` (diagnostic measurement tool, no pass/fail) from `compare_attention_output` (verifier tolerance check), and `corridor.rs` module doc explicitly states "measurement tool — no pass/fail threshold"; (4) `CorridorScaleOverrides` documented as "used only in the corridor/debug path, not in the verification protocol", and `bounds.rs` explicitly states pure replay from QKV accumulators alone cannot support ≤1 guarantee — committed intermediates are necessary.
4. [ ] Derive per-step tolerance bounds and validate empirically, done when each step of the attention chain (dequant, RoPE, Q·K score, softmax, α@V weighted sum, INT8 quantization) has both an analytical upper bound and empirical validation on the real GPU kernel. Check-and-gate with committed intermediates (KV from #3, attn_out_i8 already retained, and challenged-token Q from the opened shell/bridge path in #2) makes this a single-step problem — errors cannot compound across layers because the verifier continues from committed values. The empirical measurement must follow the same rule: compare verifier/GPU reference values against the committed/opened intermediate for that step, then continue the next step from the committed/opened value rather than from the replayed approximation. On the current committed-KV path, treat the first generated token (`token_index = n_prompt_tokens - 1`) and later decode tokens as the authoritative empirical targets. Prompt-side prefill positions (`token_index < n_prompt_tokens - 1`) are currently diagnostic only: they still lack the initial prompt KV needed for structurally complete replay, so they must not be used as headline corridor numbers. But the single-step bound still determines the accepted neighborhood and false-reject rate, so it must be tight. Explicit theorem target: derive a sufficient condition for `max_abs_diff = 1` in i8 space, i.e. prove that if the pre-quantization float error satisfies `||Δa_f||_inf < scale_a` then requantized outputs differ by at most one bucket. Current formal result: pure replay from QKV accumulators alone cannot support a worst-case `<= 1` guarantee; the theorem target is only plausible once the verifier compares against stronger committed/opened intermediates. In particular, the viable formal-`<= 1` paths are: (a) challenged-token Q from the opened shell/bridge path plus committed full-causal K/V, (b) score anchoring from #5a, or (c) commitment-bound attn_out_i8 as a weaker fallback that removes the corridor but does not independently verify attention derivation. Analytical bounds give the ceiling: score error is dominated by dequant+RoPE input perturbation (Δscore ≈ δq·k + q·δk, not just FP32 accumulator epsilon); quantization cliffs mean coordinates near INT8 bucket boundaries can flip from arbitrarily small float changes, so the margin distribution matters, not just machine epsilon; kernel-specific details (dequant semantics, RoPE implementation, accumulation order, block/group scales) affect real bounds. Empirical measurement validates: real L-inf per step, real agreement rates, false-reject rate at the analytical bound, whether the theoretical ceiling is remotely tight. To skip empirical validation entirely would require a fully specified canonical attention arithmetic path (#15), not just committed intermediates.
5a. [ ] Add score anchoring for deep audits, done when the protocol can commit and verify pre-softmax attention scores for challenged tokens. Q and K accumulators are INT32 (shell-verified); the score tolerance comes from dequant+RoPE BF16-vs-f64 perturbation on the inputs (Δscore ≈ δq·k + q·δk), not softmax amplification. This catches attention pattern manipulation (wrong K positions getting high scores) without requiring FP softmax agreement. Cost: O(seq_len × n_heads) per audited token per layer. This is now one of the explicit viable paths to a formal `<= 1` story for the checked object, unlike pure replay from sparse QKV accumulators. The score tolerance bound needs the same analytical+empirical treatment as #4.
5b. [ ] Analyze W_o conditioning for the commitment-bound security argument, done when σ_min(W_o) is computed per layer and included in the verifier key. This quantifies the adversary's freedom under routine audits where attn_out_i8 is commitment-bound but not independently replayed: given fixed next-layer input (shell-verified) and fixed W_o, the set of attn_out_i8 vectors satisfying W_o × attn_out ≈ target is bounded by the Freivalds threshold divided by σ_min. For full-rank W_o with reasonable conditioning, this is at most a few INT8 steps — essentially zero practical freedom for meaningful attention manipulation.
